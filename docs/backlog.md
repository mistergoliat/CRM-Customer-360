## Reglas operativas para el backlog

1. Si una tarea solo necesita definicion antes de codigo, mantenerla en `DESIGN`.
2. Si una tarea ya puede cambiar codigo, mantenerla en `IMPLEMENTATION`.
3. Si una tarea requiere validacion humana de criterio, mantenerla en `REVIEW`.
4. No mover tareas entre prioridades sin justificar el impacto operativo.
5. No usar este backlog para explicar el producto; usarlo para ejecutar trabajo.
